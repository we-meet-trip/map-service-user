MAP 여행 서비스의 사용자/인증 도메인 API 문서입니다. +
모든 API는 `/api/v1/auth` 경로 아래에 위치합니다.

== 공통

=== 인증 방식
모든 인증이 필요한 요청에는 `Authorization: Bearer {accessToken}` 헤더를 포함합니다.

=== 에러 응답 형식
[source,json]
----
{
  "code": "AUTH_001",
  "message": "이미 사용 중인 이메일입니다."
}
----

== 카카오 로그인 (인가 코드 방식)

`POST /api/v1/auth/kakao/callback`

Flutter SDK가 발급한 카카오 인가 코드를 서버로 전달하면, 서버가 카카오 API와 교환하여 JWT를 발급합니다. +
신규 사용자는 자동 가입되며, 이메일 동의 거부 시 `user.email` 은 null 입니다.
